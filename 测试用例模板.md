### 1.2 JSON 字段填写说明

#### headers 示例

```json
{"Content-Type": "application/json", "Authorization": "Bearer {{token}}"}
```

> **注意**：Excel 单元格中的 JSON 必须使用双引号，且为紧凑格式（无换行）。

#### body 示例

```json
{"name": "test_user", "email": "test@example.com", "role": "member"}
```

#### extract 变量提取规则

格式：`变量名=JSONPath表达式`，多个提取用 `; ` 分号加空格分隔。

```
token=$.data.token; user_id=$.data.id; user_name=$.data.name
```

支持的 JSONPath 语法：

| 表达式 | 说明 |
|---|---|
| `$.data.token` | 提取顶层 data 对象的 token |
| `$.data.list[0].id` | 提取列表第一个元素的 id |
| `$.data.list[*].name` | 提取所有元素的 name |
| `$..id` | 递归搜索所有 id 字段 |

#### assertions 断言列表

格式为 JSON 数组，支持多组断言组合：

```json
[
  {"type": "status_code", "value": 200},
  {"type": "jsonpath", "key": "$.code", "value": "0"},
  {"type": "jsonpath", "key": "$.data.token", "not_null": true},
  {"type": "response_time", "max_ms": 3000},
  {"type": "contains", "value": "success"},
  {"type": "soft", "key": "$.data.optional_field", "value": "expected", "comment": "非关键字段，失败不中断"}
]
```

#### tags 标签规范

| 标签 | 含义 | 建议使用场景 |
|---|---|---|
| `smoke` | 冒烟测试 | 核心流程，每次提交必跑 |
| `regression` | 回归测试 | 全量用例，每日/每周跑 |
| `p0` | 最高优先级 | 阻塞性 Bug 相关用例 |
| `p1` | 高优先级 | 主流程用例 |
| `p2` | 中优先级 | 边界校验用例 |
| `p3` | 低优先级 | 极端场景、兼容性 |

| 类型 | 格式 | 示例 |
|---|---|---|
| ID | `#id` | `#username` |

## 三、Excel 文件组织规范

### 3.1 文件命名

```
data/api/
  ├── 01_用户模块.xlsx          # 序号 + 模块名
  ├── 02_订单模块.xlsx
  ├── 03_商品模块.xlsx
  └── ...

data/ui/
  ├── 01_登录页.xlsx
  ├── 02_仪表盘.xlsx
  ├── 03_订单管理.xlsx
  └── ...
```

### 3.2 Sheet 组织

一个 Excel 文件可以有多个 Sheet：

```
01_用户模块.xlsx
  ├── Sheet: 登录         ← 登录相关用例
  ├── Sheet: 用户CRUD     ← 增删改查用例
  └── Sheet: 权限         ← 权限校验用例
```

**规则**：
- 每个 Sheet 第一行为表头（case_id / case_name / ...）
- 数据行从第 2 行开始
- 空行自动跳过
- 以 `#` 开头的行视为注释，自动跳过
- `skip` 列值为 `Y` 的行自动跳过

### 3.3 公共数据 Excel（变量池）

`data/common/test_data.xlsx` 用于存储跨用例的公共测试数据：

| var_name | var_value | scope | description |
|---|---|---|---|
| admin_username | admin | GLOBAL | 管理员账号 |
| admin_password | Test@123456 | GLOBAL | 管理员密码 |
| test_product_id | PROD_001 | SESSION | 测试商品ID |
| base_url | https://test-api.example.com | GLOBAL | 测试环境地址 |

> **注意**：公共数据中的敏感字段（密码等）建议通过 YAML 环境配置注入，而非直接写在 Excel 中。

## 四、内置函数速查

所有 `${函数(参数)}` 在用例执行前会被动态替换：

| 函数 | 说明 | 输出示例 |
|---|---|---|
| `${random_string(8)}` | 8 位随机字母 | `aBcDeFgH` |
| `${random_int(1,9999)}` | 1~9999 随机整数 | `5432` |
| `${timestamp()}` | 当前 Unix 时间戳（秒） | `1716681600` |
| `${timestamp_ms()}` | 当前 Unix 时间戳（毫秒） | `1716681600123` |
| `${uuid()}` | 随机 UUID | `c9bf9e57-1685-4c89-bafb-ff5af830be8a` |
| `${date_now(%Y%m%d)}` | 当前日期 | `20260526` |
| `${date_now(%Y-%m-%d %H:%M:%S)}` | 当前日期时间 | `2026-05-26 10:30:45` |
| `${phone()}` | 随机大陆手机号 | `13876543210` |

## 五、常见写法规避指南

| ❌ 错误写法 | ✅ 正确写法 | 原因 |
|---|---|---|
| `{"name":"test"}'` | `{"name":"test"}` | JSON 必须用双引号 |
| `{"name":"{{token}}"}` | 先去 `extract` 提取 token | 变量引用需要先在 extract 中定义 |
| 断言写 `code=0` | `[{"type":"jsonpath","key":"$.code","value":"0"}]` | 断言必须用 JSON 数组 |
| `NULL` / `null` / 空 | `null` 或直接不填 | Excel 中空单元格即 None |
| `$.data[0].name` | `$.data.list[0].name` | JSONPath 索引需正确 |
| `smoke P0` | `smoke,p0` | 多个标签用英文逗号分隔 |
