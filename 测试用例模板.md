### 1.2 各字段详细说明

#### headers 示例

```json
{"Content-Type": "application/json", "Authorization": "Bearer {{token}}"}
```

> **注意**：Excel 单元格中的 JSON 必须使用双引号，且为紧凑格式（无换行）。

#### params（URL 查询参数）示例

```json
{"page": 1, "size": 10, "keyword": "{{search_keyword}}"}
```

#### body 示例

```json
// JSON 格式请求体
{"name": "test_user", "email": "test@example.com", "role": "member"}

// 表单格式请求体
{"key1": "value1", "key2": "value2"}
```

> 如需发送表单数据，请将 `headers` 中 `Content-Type` 设为 `application/x-www-form-urlencoded`。

#### extract 变量提取规则

格式：`变量名=JSONPath表达式`，多个提取用 `; ` 分号加空格分隔。

```
token=$.data.token; user_id=$.data.id; user_name=$.data.name
```

支持的 JSONPath 语法：

| 表达式 | 说明 |
|---|---|
| `$.data.token` | 提取顶层 data 对象的 token |
| `$.data.list[0].id` | 提取列表第一个元素的 id |
| `$.data.list[*].name` | 提取所有元素的 name |
| `$..id` | 递归搜索所有 id 字段 |
| `$.headers['X-Request-Id']` | 提取响应头中的字段（如框架支持） |

#### 变量引用规则

| 写法 | 来源 | 作用域 | 示例 |
|---|---|---|---|
| `{{token}}` | extract 提取或 YAML 配置 | SESSION/GLOBAL | `{"Authorization":"Bearer {{token}}"}` |
| `${timestamp()}` | 内置函数 | 实时生成 | `"name":"user_${timestamp()}"` |
| `${uuid()}` | 内置函数 | 实时生成 | `"email":"${uuid()}@test.com"` |

#### assertions 断言列表

格式为 JSON 数组，支持多组断言组合：

```json
[
  {"type": "status_code", "value": 200},
  {"type": "jsonpath", "key": "$.code", "value": "0"},
  {"type": "jsonpath", "key": "$.data.token", "not_null": true},
  {"type": "response_time", "max_ms": 3000},
  {"type": "contains", "value": "success"},
  {"type": "header", "key": "Content-Type", "value": "application/json"},
  {"type": "db", "query": "SELECT count(*) FROM users WHERE email='{{test_email}}'", "value": 1},
  {"type": "soft", "key": "$.data.optional_field", "value": "expected", "comment": "非关键字段，失败不中断"}
]
```

#### tags 标签规范

| 标签 | 含义 | 建议使用场景 |
|---|---|---|
| `smoke` | 冒烟测试 | 核心流程，每次提交必跑 |
| `regression` | 回归测试 | 全量用例，每日/每周跑 |
| `p0` | 最高优先级 | 阻塞性 Bug 相关用例 |
| `p1` | 高优先级 | 主流程用例 |
| `p2` | 中优先级 | 边界校验用例 |
| `p3` | 低优先级 | 极端场景、兼容性 |

## 三、断言类型完整参考

### 3.1 常用断言

```json
// 1. 状态码断言
{"type": "status_code", "value": 200}

// 2. JSONPath 精确值
{"type": "jsonpath", "key": "$.code", "value": "0"}

// 3. JSONPath 非空
{"type": "jsonpath", "key": "$.data.token", "not_null": true}

// 4. 响应包含文本
{"type": "contains", "value": "success"}

// 5. 响应时间上限
{"type": "response_time", "max_ms": 3000}

// 6. 响应头校验
{"type": "header", "key": "Content-Type", "value": "application/json"}

// 7. JSON Schema 结构校验
{"type": "schema", "schema": {"type": "object", "required": ["code", "data"]}}

// 8. 数据库断言
{"type": "db", "query": "SELECT status FROM users WHERE id='{{user_id}}'", "value": "active"}

// 9. 软断言（失败不中断）
{"type": "soft", "key": "$.data.optional_field", "value": "expected", "comment": "非关键字段"}
```

### 3.2 断言组合示例

```json
// 登录成功：状态码 + code + token非空
[
  {"type": "status_code", "value": 200},
  {"type": "jsonpath", "key": "$.code", "value": "0"},
  {"type": "jsonpath", "key": "$.data.token", "not_null": true}
]

// 创建资源：状态码 201 + id 非空 + 数据库验证
[
  {"type": "status_code", "value": 201},
  {"type": "jsonpath", "key": "$.data.id", "not_null": true},
  {"type": "db", "query": "SELECT count(*) FROM orders WHERE id='{{order_id}}'", "value": 1}
]

// 参数校验失败：400 + 错误码 + 错误信息
[
  {"type": "status_code", "value": 400},
  {"type": "jsonpath", "key": "$.code", "value": "10001"},
  {"type": "contains", "value": "参数错误"}
]
```

## 四、变量提取与跨用例传递

### 4.1 变量提取规则

`extract` 列格式：`变量名=JSONPath表达式`

```
# 单个变量
token=$.data.token

# 多个变量（分号+空格分隔）
token=$.data.token; user_id=$.data.id; refresh_token=$.data.refresh_token
```

### 4.2 变量使用位置

变量 `{{变量名}}` 可以在以下列中使用：

| 列名 | 是否支持变量 | 示例 |
|---|---|---|
| `url` | ✅ | `/api/v1/users/{{user_id}}` |
| `headers` | ✅ | `{"Authorization":"Bearer {{token}}"}` |
| `params` | ✅ | `{"user_id":"{{user_id}}"}` |
| `body` | ✅ | `{"id":"{{user_id}}","name":"{{user_name}}"}` |
| `assertions` | ✅ | `{"type":"db","query":"SELECT * FROM users WHERE id='{{user_id}}'"}` |

### 4.3 用例依赖示例

```
# API_USER_001（无依赖）
extract: token=$.data.token
  ↓ depends_on
# API_USER_002（依赖 API_USER_001）
extract: user_id=$.data.id
  ↓ depends_on
# API_USER_003（依赖 API_USER_002）
url: /api/v1/users/{{user_id}}
```

## 五、钩子使用模板

### 5.1 前置钩子 (pre_hook)

在 Excel `pre_hook` 列填写钩子函数名：

| case_id | case_name | method | url | pre_hook | ... |
|---|---|---|---|---|---|
| API_USER_001 | 正常登录 | POST | /api/v1/login | login_as_admin | ... |
| API_ORDER_001 | 查询订单 | GET | /api/v1/orders | login_as_admin, init_order_data | ... |

> 多个钩子用英文逗号 `,` 分隔。

钩子函数定义（`hooks/sample_hooks.py`）：

```python
def login_as_admin(client, variable_manager, db=None):
    """管理员登录，注入 token"""
    resp = client.post("/api/v1/login", json={
        "username": "admin",
        "password": "Test@123"
    })
    token = resp.json()["data"]["token"]
    variable_manager.set("token", token)
    client.session.headers["Authorization"] = f"Bearer {token}"
```

### 5.2 后置钩子 (post_hook)

在 Excel `post_hook` 列填写钩子函数名：

| case_id | case_name | method | url | post_hook | ... |
|---|---|---|---|---|---|
| API_USER_002 | 创建用户 | POST | /api/v1/users | cleanup_user | ... |

钩子函数定义：

```python
def cleanup_user(client, variable_manager, db=None, result=None):
    """清理测试数据"""
    user_id = variable_manager.SESSION.get("user_id")
    if user_id and db:
        db.execute(f"DELETE FROM users WHERE id = {user_id}")
```

## 六、Excel 文件组织规范

### 6.1 文件命名

```
data/api/
  ├── 01_用户模块.xlsx          # 序号 + 模块名
  ├── 02_订单模块.xlsx
  ├── 03_商品模块.xlsx
  ├── 04_支付模块.xlsx
  └── ...
```

### 6.2 Sheet 组织

一个 Excel 文件可以有多个 Sheet：

```
01_用户模块.xlsx
  ├── Sheet: 登录注册        ← 登录注册相关用例
  ├── Sheet: 用户CRUD        ← 增删改查用例
  └── Sheet: 权限校验        ← 权限校验用例
```

**规则**：
- 每个 Sheet 第一行为表头（case_id / case_name / ...）
- 数据行从第 2 行开始
- 空行自动跳过
- 以 `#` 开头的行视为注释，自动跳过
- `skip` 列值为 `Y` 的行自动跳过

### 6.3 公共数据管理

公共数据建议通过 YAML 环境配置注入（`config/env/{env}.yaml` 中的 `custom` 字段）：

```yaml
# config/env/dev.yaml
custom:
  admin_username: "admin"
  admin_password: "${ADMIN_PASSWORD}"
  test_product_id: "PROD_001"
```

在 Excel 中通过 `{{变量名}}` 引用：

```
body: {"username":"{{admin_username}}","password":"{{admin_password}}"}
```

## 七、内置函数速查

所有 `${函数(参数)}` 在用例执行前会被动态替换：

| 函数 | 说明 | 输出示例 |
|---|---|---|
| `${random_string(8)}` | 8 位随机字母 | `aBcDeFgH` |
| `${random_int(1,9999)}` | 1~9999 随机整数 | `5432` |
| `${timestamp()}` | 当前 Unix 时间戳（秒） | `1716681600` |
| `${timestamp_ms()}` | 当前 Unix 时间戳（毫秒） | `1716681600123` |
| `${uuid()}` | 随机 UUID | `c9bf9e57-1685-4c89-bafb-ff5af830be8a` |
| `${date_now(%Y%m%d)}` | 当前日期 | `20260526` |
| `${date_now(%Y-%m-%d %H:%M:%S)}` | 当前日期时间 | `2026-05-26 10:30:45` |
| `${phone()}` | 随机大陆手机号 | `13876543210` |

## 八、常见写法规避指南

| ❌ 错误写法 | ✅ 正确写法 | 原因 |
|---|---|---|
| `{"name":"test"'}`  | `{"name":"test"}` | JSON 必须用双引号 |
| `{"name":"{{token}}"}` | 先去 `extract` 提取 token | 变量引用需要先在 extract 中定义 |
| 断言写 `code=0` | `[{"type":"jsonpath","key":"$.code","value":"0"}]` | 断言必须用 JSON 数组 |
| `NULL` / `null` / 空 | `null` 或直接不填 | Excel 中空单元格即不传该字段 |
| `$.data[0].name` | `$.data.list[0].name` | JSONPath 索引需正确 |
| `smoke P0` | `smoke,p0` | 多个标签用英文逗号分隔 |
| `username: {{admin_username}}` | 在 URL 中用到变量需要 URL 编码注意 | 大多数框架会自动 URL 编码 |
