## 1. 设计目标

| 目标 | 说明 |
|---|---|
| **零代码编写** | 测试工程师只需在 Excel 中填写用例，无需写 Python |
| **数据驱动** | 用例数据（URL、参数、断言）全部由 Excel 驱动 |
| **变量引擎** | 支持跨用例变量传递，实现用例依赖编排 |
| **多环境切换** | 通过 `--env` 参数一键切换 dev/test/prod |
| **可扩展** | 钩子系统支持自定义鉴权、业务前置/后置逻辑 |
| **CI/CD 就绪** | 支持并行执行、失败重跑、Allure 报告 |

## 3. 数据流与执行链路

```
Excel 文件 (data/api/*.xlsx)
    │
    ▼
ExcelReader 逐 Sheet 逐行读取
    │
    ▼
CaseParser 解析每行为 ApiCaseModel
    │
    ▼
DataDriver 将 ApiCaseModel 列表参数化到 pytest
    │
    ▼
pytest 逐一执行每个用例 ──────┐
    │                         │
    ▼                         │
ApiTestEngine.run(case)      │
    │                         │
    ├─ 1. 变量替换             │
    ├─ 2. 执行 pre_hook        │
    ├─ 3. 发送 HTTP 请求       │
    ├─ 4. 提取响应变量 (extract)│
    ├─ 5. 执行断言 (assertions) │
    └─ 6. 执行 post_hook       │
    │                         │
    ▼                         │
TestResult (passed/failed)   │
    │                         │
    ▼                         │
Allure 报告                   │
                              │
    依赖解析 ←─────────────────┘
    (depends_on 字段决定执行顺序)
```

### 变量生命周期

```
Session 启动 → VariableManager 初始化 → GLOBAL 变量加载
    │
    │ 每个用例执行时:
    ├─ CASE 变量注入（内置函数解析）
    ├─ 依赖变量注入（depends_on 链路传递）
    ├─ extract 提取 → SESSION 变量
    └─ 用例结束 → CASE 变量回收

Session 结束 → SESSION 变量清空
```

## 4. Excel 用例模型

### 4.1 列定义

| 列名 | 必填 | 类型 | 说明 |
|---|---|---|---|
| `case_id` | ✅ | string | 唯一标识，如 `API_USER_001` |
| `case_name` | ✅ | string | 用例名称 |
| `module` | ❌ | string | 模块名（分组用） |
| `method` | ✅ | string | HTTP 方法：GET/POST/PUT/DELETE/PATCH |
| `url` | ✅ | string | 接口路径，如 `/api/v1/users` |
| `headers` | ❌ | JSON | 请求头 |
| `params` | ❌ | JSON | URL 查询参数 |
| `body` | ❌ | JSON/string | 请求体 |
| `extract` | ❌ | string | 变量提取规则，如 `token=$.data.token; user_id=$.data.id` |
| `assertions` | ✅ | JSON Array | 断言列表 |
| `depends_on` | ❌ | string | 依赖上游 case_id（逗号分隔） |
| `pre_hook` | ❌ | string | 前置钩子函数名 |
| `post_hook` | ❌ | string | 后置钩子函数名 |
| `skip` | ❌ | Y/N | 是否跳过 |
| `tags` | ❌ | string | 标签：smoke,regression,p0,p1,p2（逗号分隔） |

### 4.2 数据模型

```python
@dataclass
class ApiCaseModel:
    case_id: str
    case_name: str
    module: str = ""
    method: str = "GET"
    url: str = ""
    headers: dict = field(default_factory=dict)
    params: dict = field(default_factory=dict)
    body: Any = None
    extract: Dict[str, str] = field(default_factory=dict)   # {变量名: JSONPath}
    assertions: List[AssertionItem] = field(default_factory=list)
    depends_on: List[str] = field(default_factory=list)
    pre_hook: str = ""
    post_hook: str = ""
    skip: bool = False
    tags: List[str] = field(default_factory=list)
```

## 5. 用例解析器 (CaseParser)

```python
class CaseParser:
    def parse_api_case(self, row: dict) -> ApiCaseModel:
        """将 Excel 一行数据转为 ApiCaseModel"""

    def parse_assertions(self, raw: str) -> List[AssertionItem]:
        """解析断言 JSON 数组"""

    def parse_extract(self, raw: str) -> Dict[str, str]:
        """解析 extract 字段:
        'token=$.data.token; user_id=$.data.id'
        → {'token': '$.data.token', 'user_id': '$.data.id'}
        """

    def parse_tags(self, raw: str) -> List[str]:
        """解析 tags 列, 逗号分隔"""
```

解析时会对 JSON 字段（headers / params / body / assertions）做容错处理，自动修复常见格式错误（如单引号转双引号）。

## 6. 变量引擎 (VariableManager)

### 6.1 三级作用域

| 作用域 | 键 | 来源 | 生命周期 |
|---|---|---|---|
| **GLOBAL** | `VariableManager.GLOBAL` | YAML 全局配置 | 整个 session |
| **SESSION** | `VariableManager.SESSION` | extract + 依赖传递 | 整个 session |
| **CASE** | `VariableManager.CASE` | 内置函数 + 当前用例上下文 | 单个用例 |

查找顺序：CASE → SESSION → GLOBAL

### 6.2 变量引用语法

| 写法 | 说明 | 示例 |
|---|---|---|
| `{{token}}` | 从变量池获取 `token` | `{"Authorization":"Bearer {{token}}"}` |
| `${timestamp()}` | 调用内置函数 | `"name":"user_${timestamp()}"` |
| `${var:default}` | 带默认值 | `"host":"${DB_HOST:localhost}"` |

### 6.3 内置函数

| 函数 | 说明 | 输出示例 |
|---|---|---|
| `timestamp()` | 当前 Unix 时间戳（秒） | `1716681600` |
| `timestamp_ms()` | 当前毫秒时间戳 | `1716681600123` |
| `uuid()` | UUID v4 | `c9bf9e57-...` |
| `random_string(n)` | n 位随机字母 | `aBcDeFgH` |
| `random_int(a,b)` | a~b 随机整数 | `5432` |
| `date_now(format)` | 日期格式化 | `2026-05-26` |
| `phone()` | 随机手机号 | `13876543210` |

### 6.4 变量替换流程

```python
def replace_variables(self, obj: Any) -> Any:
    """
    递归替换对象中的 {{var}} 和 ${func()} 占位符。
    对于字符串 → 替换后返回
    对于 dict → 递归替换 value
    对于 list → 递归替换元素
    """
```

## 7. HTTP 引擎 (ApiTestEngine)

```python
class ApiTestEngine:
    def __init__(self, session: SessionManager, db_helper=None):
        self.session = session        # requests.Session 封装
        self.base_url = session.base_url
        self.db_helper = db_helper

    def run(self, case: ApiCaseModel) -> TestResult:
        """单用例完整执行流程:
        1. 替换变量
        2. 执行前置钩子
        3. 构造请求 → 发送 → 计时
        4. 提取响应变量
        5. 执行断言链
        6. 执行后置钩子
        7. 返回 TestResult
        """

    def _send_request(self, case: ApiCaseModel) -> requests.Response:
        """构造并发送 HTTP 请求:
        - 拼接 base_url + url
        - 应用 headers/params/body
        - 支持 retry 重试
        - 记录 request/response 到 TestResult
        """

    def _extract_variables(self, case, response):
        """根据 extract 规则从响应中提取变量:
        - 遍历 case.extract
        - jsonpath-ng 解析响应 JSON
        - 将提取值写入 VariableManager.SESSION
        """
```

## 8. 断言引擎 (AssertionEngine)

```python
class AssertionEngine:
    def run_all(self,
                response: requests.Response,
                assertions: List[AssertionItem],
                db_helper=None,
                response_time_ms: float = 0
    ) -> List[dict]:
        """执行全部断言，返回每条的 {type, passed, actual, expected, message}"""

    def _assert_status_code(self, response, assertion): ...
    def _assert_jsonpath(self, response, assertion): ...
    def _assert_contains(self, response, assertion): ...
    def _assert_response_time(self, response_time_ms, assertion): ...
    def _assert_header(self, response, assertion): ...
    def _assert_schema(self, response, assertion): ...
    def _assert_db(self, db_helper, assertion): ...
    def _assert_soft(self, response, assertion): ...
```

### 断言类型完整参考

| type | 参数 | 说明 |
|---|---|---|
| `status_code` | `value` (int) | HTTP 状态码校验 |
| `jsonpath` | `key` (JSONPath), `value` 或 `not_null` | JSON 字段值/非空校验 |
| `contains` | `value` (string) | 响应体包含指定文本 |
| `response_time` | `max_ms` (int) | 响应时间上限 |
| `header` | `key`, `value` | 响应头校验 |
| `schema` | `schema` (JSON Schema) | JSON Schema 结构校验 |
| `db` | `query` (SQL), `value` | 数据库断言 |
| `soft` | 同 jsonpath + `comment` | 软断言，失败不中断 |

### 断言执行流程

```
响应 status_code → jsonpath → contains → response_time → header → schema → db
     │                │
     │  失败           │  失败
     ▼                ▼
  立即中断        根据 mode: hard=中断 / soft=记录继续
```

## 10. 钩子系统 (HookManager)

```python
class HookManager:
    _pre_hooks: Dict[str, Callable] = {}
    _post_hooks: Dict[str, Callable] = {}

    @classmethod
    def register(cls, name: str, func: Callable, hook_type: str):
        """注册钩子函数"""

    @classmethod
    def register_from_module(cls, module_path: str):
        """从 Python 模块自动注册钩子"""

    @classmethod
    def run_pre_hook(cls, name: str, client, variable_manager, db_helper):
        """执行前置钩子"""

    @classmethod
    def run_post_hook(cls, name: str, client, variable_manager, db_helper, result):
        """执行后置钩子"""
```

### 钩子函数签名规范

```python
# hooks/sample_hooks.py

def login_as_admin(client, variable_manager, db=None):
    """前置钩子示例：管理员登录获取 token"""
    resp = client.post("/api/v1/login", json={
        "username": "admin",
        "password": "Test@123"
    })
    token = resp.json()["data"]["token"]
    variable_manager.set("token", token)
    # 自动将 token 注入后续请求头
    client.session.headers["Authorization"] = f"Bearer {token}"


def cleanup_user(client, variable_manager, db=None, result=None):
    """后置钩子示例：清理测试创建的用户"""
    user_id = variable_manager.SESSION.get("user_id")
    if user_id and db:
        db.execute(f"DELETE FROM users WHERE id = {user_id}")
```

Excel 中钩子的使用：
- `pre_hook` 列填 `login_as_admin`
- `post_hook` 列填 `cleanup_user`
- 多个钩子用逗号分隔：`login_as_admin, init_test_data`

## 14. 用例依赖与编排

通过 `depends_on` 字段声明用例间数据依赖：

```
用例 A (depends_on=None)
    POST /api/v1/login → 提取 token
     ↓
用例 B (depends_on=API_USER_001)
    POST /api/v1/users → 使用 token + 提取 user_id → 提取 user_id
     ↓
用例 C (depends_on=API_USER_002)
    GET /api/v1/users/{{user_id}} → 使用 token + user_id
```

## 15. 数据清理保障

使用 `try-finally` 保证后置钩子一定执行：

```python
class ApiTestEngine:
    def run(self, case: ApiCaseModel) -> TestResult:
        result = TestResult(...)
        try:
            self._send_request(case, result)
            self._execute_assertions(case, result)
            result.passed = True
        except Exception as e:
            result.error_message = str(e)
            raise
        finally:
            try:
                self._execute_post_hook(case)
            except Exception as e:
                logger.warning(f"后置钩子执行失败: {e}")
        return result
```

### 数据清理策略

| 策略 | 适用场景 |
|---|---|
| `post_hook` 指定清理 | 用例级精准清理 |
| 变量追踪清理 | 自动清理用例创建的测试数据 |
| 时间戳隔离 | 数据带前缀+时间戳，定时任务批量清理 |

## 16. 异常处理体系
