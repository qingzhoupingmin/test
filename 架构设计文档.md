# pytest + requests + playwright + yaml + allure 通用自动化测试框架架构设计文档

---

## 1. 项目概述

### 1.1 目标

构建一套 **Excel 驱动** 的通用自动化测试框架，统一支撑 **接口测试（API）** 与 **UI 测试**，实现零代码/低代码的用例管理与执行。

### 1.2 核心理念

> **用例即数据，数据即用例** —— 测试人员只需在 Excel 中维护用例数据，框架自动解析、执行并生成 Allure 报告。

### 1.3 技术栈

| 组件 | 技术选型 | 用途 |
|---|---|---|
| 测试框架 | `pytest` | 用例调度、fixture 管理、参数化 |
| 接口测试 | `requests` | HTTP 请求发送、会话管理 |
| UI 测试 | `playwright` | 浏览器自动化、页面操作 |
| 数据配置 | `yaml` | 环境配置、全局变量、数据库配置 |
| 用例数据 | `openpyxl` / `pandas` | 解析 Excel 用例文件 |
| 报告 | `allure-pytest` | 测试报告生成与可视化 |
| 断言 | 内置 + `jsonpath` / `jsonschema` | 接口响应断言 + UI 状态断言 |
| 日志 | `loguru` | 结构化日志输出 |

### 3.2 数据流设计

```
Excel 文件
    │
    ▼
[ExcelReader]  读取 Excel → List[Dict]
    │
    ▼
[CaseParser]   解析每行 → CaseModel 对象列表
    │
    ▼
[VariableManager]  处理变量引用 ({{var}})、提取函数调用 (${func()})
    │
    ▼
[DataDriver]   转换为 pytest parametrize 数据 → (case_name, case_data)[]
    │
    ▼
[pytest]       根据 case_data.type 分发到 API Runner 或 UI Runner
    │
    ├──► [API Runner]  发送 HTTP 请求 → 响应
    │        │
    │        ▼
    │    [AssertionEngine]  断言校验（status_code / jsonpath / schema / db）
    │
    ├──► [UI Runner]   执行浏览器操作 → 页面状态
    │        │
    │        ▼
    │    [AssertionEngine]  断言校验（元素可见 / 文本匹配 / 页面URL）
    │
    ▼
[Allure]       生成测试报告（步骤、截图、日志、请求/响应详情）
```

### 6.4 AssertionEngine — 断言引擎

```python
# core/assertion_engine.py
from typing import Any, Dict, List

class AssertionEngine:
    """
    统一断言引擎，支持硬断言 + 软断言
    硬断言：失败立即抛异常
    软断言：收集所有失败，用例结束时统一报告
    """

    def __init__(self):
        self._soft_errors: List[str] = []   # 软断言错误收集

    # ============= 硬断言 =============

    def assert_status_code(self, actual: int, expected: int):
        """HTTP 状态码断言（硬）"""
        pass

    def assert_jsonpath(self, response_json: Dict, jsonpath_expr: str, expected: Any):
        """JSONPath 断言（硬）"""
        pass

    def assert_json_schema(self, response_json: Dict, schema: Dict):
        """JSON Schema 校验（硬）"""
        pass

    def assert_contains(self, response_text: str, expected: str):
        """响应包含断言（硬）"""
        pass

    def assert_header(self, headers: Dict, key: str, expected: str):
        """响应头断言（硬）"""
        pass

    def assert_response_time(self, elapsed: float, max_ms: float):
        """响应时间断言（硬）"""
        pass

    def assert_db(self, query: str, expected: Any):
        """数据库断言（硬）"""
        pass

    # ============= 软断言 =============

    def soft_assert(self, condition: bool, message: str):
        """软断言：记录失败但不中断，用例结束后统一抛出"""
        if not condition:
            self._soft_errors.append(message)

    def assert_all_soft(self):
        """用例结束时调用，汇集所有软断言失败并抛出"""
        if self._soft_errors:
            raise AssertionError(f"软断言失败 {len(self._soft_errors)} 条:\n" + "\n".join(self._soft_errors))

    # ============= UI 断言 =============

    def assert_element_visible(self, page, selector: str):
        """元素可见断言（硬）"""
        pass

    def assert_text_equals(self, page, selector: str, expected: str):
        """文本匹配断言（硬）"""
        pass

    def assert_url_contains(self, page, expected: str):
        """URL 包含断言（硬）"""
        pass

    def assert_page_title(self, page, expected: str):
        """页面标题断言（硬）"""
        pass
```

#### 断言记录格式（Excel `assertions` 列）

```json
[
  {"type": "status_code", "value": 200},
  {"type": "jsonpath", "key": "$.code", "value": "0"},
  {"type": "jsonpath", "key": "$.data.token", "not_null": true},
  {"type": "contains", "value": "success"},
  {"type": "response_time", "max_ms": 3000},
  {"type": "soft", "key": "$.data.unused_field", "value": null, "comment": "软断言-非关键字段"}
]
```

**断言字段说明：**

| 断言 type | 必填参数 | 可选参数 | 说明 |
|---|---|---|---|
| `status_code` | `value` | — | 断言 HTTP 状态码 |
| `jsonpath` | `key` | `value`, `not_null` | JSONPath 提取后比对 |
| `contains` | `value` | — | 响应体包含指定字符串 |
| `response_time` | `max_ms` | — | 响应时间上限 |
| `header` | `key`, `value` | — | 响应头断言 |

## 15. 鉴权抽象层

小型 Web 项目常见的鉴权方式统一抽象：

```python
# api/api_base.py

from abc import ABC, abstractmethod
import hashlib, hmac, time, base64

class AuthBase(ABC):
    """鉴权基类"""

    @abstractmethod
    def apply(self, session, request_args: dict) -> dict:
        """对请求参数注入鉴权信息，返回修改后的 request_args"""
        pass


class BearerTokenAuth(AuthBase):
    """JWT / Bearer Token: 从变量池取 {{token}}"""
    def __init__(self, token: str):
        self.token = token

    def apply(self, session, request_args):
        request_args.setdefault("headers", {})["Authorization"] = f"Bearer {self.token}"
        return request_args


class ApiKeyAuth(AuthBase):
    """API Key: 注入 header 或 query"""
    def __init__(self, key: str, value: str, location: str = "header"):
        self.key = key
        self.value = value
        self.location = location

    def apply(self, session, request_args):
        if self.location == "header":
            request_args.setdefault("headers", {})[self.key] = self.value
        else:
            request_args.setdefault("params", {})[self.key] = self.value
        return request_args


class SignatureAuth(AuthBase):
    """HMAC-SHA256 签名（常见于内部 API / 支付回调验签）"""
    def __init__(self, app_id: str, secret: str):
        self.app_id = app_id
        self.secret = secret

    def apply(self, session, request_args):
        timestamp = str(int(time.time()))
        body_str = json.dumps(request_args.get("body", {}) or {}, sort_keys=True)
        sign_str = f"{self.app_id}\n{timestamp}\n{body_str}"
        signature = hmac.new(self.secret.encode(), sign_str.encode(), hashlib.sha256).hexdigest()
        request_args.setdefault("headers", {})["X-App-Id"] = self.app_id
        request_args.setdefault("headers", {})["X-Timestamp"] = timestamp
        request_args.setdefault("headers", {})["X-Signature"] = signature
        return request_args


class CookieSessionAuth(AuthBase):
    """Session Cookie: 从变量池取登录后的 cookies"""
    def __init__(self, cookies: dict):
        self.cookies = cookies

    def apply(self, session, request_args):
        for k, v in self.cookies.items():
            session.cookies.set(k, v)
        return request_args
```

**Excel 中使用方式**：在 `pre_hook` 中指定鉴权钩子（如 `login_with_signature`），钩子内部实例化对应 AuthBase 子类并挂载到 HttpClient，后续用例自动携带鉴权信息。

```python
# utils/config_loader.py
import os, re, yaml

class ConfigLoader:
    ENV_PATTERN = re.compile(r'\$\{(.+?)(?::(.+?))?\}')

    @classmethod
    def load(cls, path: str) -> dict:
        raw = yaml.safe_load(open(path, encoding="utf-8"))
        return cls._resolve_env(raw)

    @classmethod
    def _resolve_env(cls, obj):
        """递归替换 ${VAR:default} 占位符"""
        if isinstance(obj, str):
            m = cls.ENV_PATTERN.fullmatch(obj)
            if m:
                env_var, default = m.group(1), m.group(2)
                return os.environ.get(env_var, default)
            return obj
        if isinstance(obj, dict):
            return {k: cls._resolve_env(v) for k, v in obj.items()}
        if isinstance(obj, list):
            return [cls._resolve_env(v) for v in obj]
        return obj
```

```python
# utils/crypto_helper.py

SENSITIVE_KEYS = {"password", "secret", "token", "access_token", "api_key"}

def mask_sensitive(data: dict) -> dict:
    """将敏感字段值替换为 ******，用于日志和报告输出"""
    if not isinstance(data, dict):
        return data
    masked = {}
    for k, v in data.items():
        if k.lower() in SENSITIVE_KEYS and isinstance(v, str):
            masked[k] = "******"
        elif isinstance(v, dict):
            masked[k] = mask_sensitive(v)
        else:
            masked[k] = v
    return masked
```

## 18. 数据清理保障

避免测试数据残留，使用 `try-finally` 保障清理一定执行：

```python
# api/api_runner.py

class ApiRunner:
    def run(self, case: ApiCaseModel) -> TestResult:
        result = TestResult(case_id=case.case_id, case_name=case.case_name, passed=False)
        try:
            self._execute_request(case, result)
            self._execute_assertions(case, result)
            result.passed = True
        except Exception as e:
            result.error_message = str(e)
            raise
        finally:
            # finally 块确保无论成功失败都执行清理钩子
            try:
                self._execute_post_hook(case)
            except Exception as hook_error:
                logger.warning(f"后置钩子执行失败: {hook_error}")
        return result
```

### 数据清理策略

| 策略 | 说明 | 适用场景 |
|---|---|---|
| **硬编码清理** | `post_hook` 中写死 DELETE SQL | 简单场景 |
| **变量追踪清理** | 步骤中标记 `{{created_user_id}}`，后置钩子自动删除 | 推荐，通用 |
| **事务回滚** | DB 操作包裹事务，用例结束时自动回滚 | 只读验证场景 |
| **时间戳隔离** | 测试数据带 `test_` 前缀 + 时间戳，定时任务批量清理 | 高频写入场景 |

## 20. 异常处理体系

```python
# core/exceptions.py

class FrameworkError(Exception):
    """框架基础异常"""
    def __init__(self, message: str, error_code: str = "FRAMEWORK_ERROR"):
        self.error_code = error_code
        super().__init__(message)


class ConfigError(FrameworkError):
    """配置错误：YAML 缺失字段、环境变量未设置"""
    def __init__(self, message: str):
        super().__init__(message, "CONFIG_ERROR")


class ParseError(FrameworkError):
    """用例解析错误：Excel 格式错误、JSON 解析失败"""
    def __init__(self, message: str, case_id: str = ""):
        self.case_id = case_id
        super().__init__(f"[{case_id}] {message}", "PARSE_ERROR")


class ExecutionError(FrameworkError):
    """执行错误：HTTP 超时、连接失败、元素未找到"""
    def __init__(self, message: str, case_id: str = ""):
        self.case_id = case_id
        super().__init__(f"[{case_id}] {message}", "EXECUTION_ERROR")


class AssertionError(FrameworkError):
    """断言失败"""
    def __init__(self, message: str, assertion_type: str = ""):
        self.assertion_type = assertion_type
        super().__init__(message, "ASSERTION_ERROR")


class VariableNotResolvedError(FrameworkError):
    """变量未解析：{{xxx}} 在变量池中找不到"""
    def __init__(self, variable_name: str):
        super().__init__(f"变量 '{{{variable_name}}}' 未找到", "VARIABLE_NOT_RESOLVED")
```

## 23. Allure 环境信息注入

在测试开始前自动写入环境元信息到 Allure 结果中：

```python
# conftest.py 中

def pytest_sessionstart(session):
    """
    测试会话开始时，注入环境信息到 allure-results 目录
    """
    import os, platform, subprocess
    from datetime import datetime

    env = session.config.getoption("--env", default="test")
    allure_dir = os.path.join("reports", "allure-results")
    os.makedirs(allure_dir, exist_ok=True)

    # 获取 git 信息
    def git_info(cmd):
        try:
            return subprocess.check_output(["git"] + cmd, stderr=subprocess.DEVNULL).decode().strip()
        except Exception:
            return "unknown"

    properties = {
        "Environment": env,
        "BaseURL": os.environ.get("BASE_URL", ""),
        "Python": platform.python_version(),
        "Platform": platform.platform(),
        "Branch": git_info(["rev-parse", "--abbrev-ref", "HEAD"]),
        "Commit": git_info(["rev-parse", "--short", "HEAD"]),
        "Executor": os.environ.get("USER", os.environ.get("USERNAME", "local")),
        "ReportTime": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
    }

    # 写入 environment.properties
    with open(os.path.join(allure_dir, "environment.properties"), "w", encoding="utf-8") as f:
        for k, v in properties.items():
            f.write(f"{k}={v}\n")

    # 写入 executor.json（CI 信息）
    import json
    executor_info = {
        "name": "Local",
        "type": "local",
        "buildName": f"Local-{datetime.now().strftime('%Y%m%d%H%M%S')}",
    }
    with open(os.path.join(allure_dir, "executor.json"), "w", encoding="utf-8") as f:
        json.dump(executor_info, f, indent=2)
```

生成的 `environment.properties` 将在 Allure 报告的 **Environment** 面板中展示环境、分支、提交哈希等信息。

## 24. 小型 Web 项目复用方案

### 24.1 适配小型项目的核心诉求

小型 Web 项目（1~5 人团队、2~20 个 API、3~10 个页面）对测试框架的要求不同于大型项目：

| 诉求 | 大型项目特征 | 小型项目特征 | 框架应对 |
|---|---|---|---|
| **学习成本** | 可投入专项人力学习 | 必须 10 分钟内上手 | 预设模板 + 可视化 Excel |
| **部署复杂度** | 可依赖 CI/CD 完整链路 | 本地一键执行 | `run.bat` / `run.sh` 开箱即用 |
| **依赖数量** | MySQL + Redis + Kafka + ES | 通常只要 HTTP + SQLite | 按需安装，数据库层可选 |
| **配置量** | 多环境复杂配置 | 可能只有一个环境 | 单文件 `config.yaml` 最小模式 |
| **用例数量** | 500+ 条 | 20~100 条 | 单个 Excel 文件即可组织全部用例 |
| **框架升级** | 专人维护升级 | 不能频繁侵入式改动 | Git Submodule 共享核心 |

### 24.2 框架与被测项目的集成方式

#### 方式一：独立目录模式（推荐小型项目首选）

框架作为独立目录放在被测项目根目录，互不侵入：

```
my-web-app/                       ← 被测项目根目录
├── backend/                      ← 后端代码
├── frontend/                     ← 前端代码
├── tests/                        ← 测试框架目录 (= 本框架)
│   ├── conftest.py
│   ├── pytest.ini
│   ├── core/                     ← 框架核心（不变）
│   ├── config/
│   │   └── env/
│   │       └── dev.yaml          ← 只改这里和下面的 Excel
│   └── data/
│       └── api/
│           └── cases.xlsx        ← 用例全在这一个文件
├── run_tests.bat                 ← 双击执行
└── docker-compose.yml
```

**优点**：框架核心代码零修改，只改 Excel 和 YAML。

#### 方式二：Git Submodule 共享核心（多项目复用）

```

└── qa-framework/
    ├── core/                     ← 核心引擎（共享，不可修改）
    ├── api/                      ← HTTP 层
    ├── ui/                       ← UI 层
    └── utils/

project-a/
├── ...
└── tests/
    ├── framework/                ← git submodule → qa-framework
    ├── config/                   ← 项目独有
    └── data/                     ← 项目独有

project-b/
└── tests/
    ├── framework/                ← git submodule → qa-framework（同一份核心）
    ├── config/
    └── data/
```

```bash

```

**优点**：框架核心升级时，`git submodule update --remote` 即可同步，各项目用例不受影响。

#### 方式三：pip 包模式（团队内部发布）


### 24.8 单项目单文件模式（极简模式）

对于仅 5~15 个接口的超小项目，可完全省略 `test_cases/` 目录，直接在项目根目录写一个 Python 测试文件：

```python
# tests/test_all.py  ← 一个文件跑完全部测试

import pytest
from pathlib import Path

# 自动发现 data/api/ 下所有 Excel
excel_files = list(Path("data/api").glob("*.xlsx"))

@pytest.mark.parametrize("excel_file", excel_files)
def test_all_cases(excel_file):
    reader = ExcelReader()
    parser = CaseParser()
    runner = ApiRunner()

    for sheet_name, rows in reader.read_all_sheets(excel_file).items():
        for row in rows:
            case = parser.parse_api_case(row)
            result = runner.run(case)
            assert result.passed, f"{case.case_id} 失败: {result.error_message}"
```

**优点**：零目录结构要求，一个文件 + 一个 Excel 即可工作。

## 25. 通知集成

测试完成后自动推送结果到 IM 工具（小型项目常用钉钉/企微群通知）：

```python
# utils/notifier.py

import requests, json

class Notifier:
    """测试结果通知"""

    def __init__(self, webhook_url: str):
        self.webhook_url = webhook_url

    def send_dingtalk(self, title: str, passed: int, failed: int, report_url: str):
        """钉钉群机器人通知"""
        payload = {
            "msgtype": "markdown",
            "markdown": {
                "title": title,
                "text": (
                    f"## {title}\n\n"
                    f"- ✅ 通过: **{passed}**\n"
                    f"- ❌ 失败: **{failed}**\n"
                    f"- 📊 通过率: **{passed/(passed+failed)*100:.1f}%**\n\n"
                    f"[查看报告]({report_url})"
                )
            }
        }
        requests.post(self.webhook_url, json=payload)

    def send_wecom(self, title: str, passed: int, failed: int, report_url: str):
        """企业微信机器人通知"""
        payload = {
            "msgtype": "markdown",
            "markdown": {
                "content": (
                    f"## {title}\n"
                    f"> ✅ 通过: <font color=\"info\">{passed}</font>\n"
                    f"> ❌ 失败: <font color=\"warning\">{failed}</font>\n"
                    f"[查看报告]({report_url})"
                )
            }
        }
        requests.post(self.webhook_url, json=payload)
```

在 `conftest.py` 中注册 session 结束钩子：

```python
# conftest.py

def pytest_sessionfinish(session, exitstatus):
    """测试结束时发送通知"""
    webhook = os.environ.get("NOTIFY_WEBHOOK")
    if not webhook:
        return

    stats = session.config.pluginmanager.get_plugin("terminalreporter").stats
    passed = len(stats.get("passed", []))
    failed = len(stats.get("failed", []))

    notifier = Notifier(webhook)
    notifier.send_dingtalk(
        title=f"自动化测试完成 [{session.config.getoption('--env')}]",
        passed=passed,
        failed=failed,
        report_url=os.environ.get("REPORT_URL", "")
    )
```

**YAML 中配置通知**：
```yaml
# config/settings.yaml
notification:
  dingtalk_webhook: "${DINGTALK_WEBHOOK:}"   # 不为空则启用
  wecom_webhook: "${WECOM_WEBHOOK:}"
  on_failure_only: true                      # 仅失败时通知
```

### 26. 版本管理与升级策略

| 模块 | 版本策略 | 说明 |
|---|---|---|
| `core/` | **严格锁定** | 框架核心，由 QA 架构师维护，各项目通过 submodule 锁定版本 |

1. QA 架构师在框架仓库发布新版本
2. 各项目执行 `git submodule update --remote tests/framework`
