## 1. 环境准备

### 1.1 Python 环境

```bash
# 确认 Python 版本 >= 3.9
python --version

# 建议使用虚拟环境
python -m venv venv

# Windows 激活
venv\Scripts\activate

# macOS/Linux 激活
source venv/bin/activate
```

### 1.2 安装依赖

```bash
pip install -r requirements.txt
```

### 1.3 安装 Playwright 浏览器

```bash
# 安装 Chromium（推荐首选）
playwright install chromium

# 如需 Firefox
playwright install firefox

# 如需 WebKit（仅 macOS）
playwright install webkit

# 安装系统依赖（Linux 服务器上必需）
playwright install-deps chromium
```

### 1.4 验证安装

```bash
pytest --version
# 输出: pytest 7.x.x

python -c "from playwright.sync_api import sync_playwright; print('OK')"
# 输出: OK

python -c "import allure; print('OK')"
# 输出: OK
```

## 2. 项目初始化

### 2.1 克隆项目

```bash
git clone <仓库地址> 通用测试框架
cd 通用测试框架
```

### 2.2 目录确认

```
通用测试框架/
├── config/          ← 环境配置文件
├── data/            ← Excel 用例文件（你主要操作这里）
│   ├── api/         ← 放 API 用例
│   ├── ui/          ← 放 UI 用例
│   └── common/      ← 放公共变量数据
├── test_cases/      ← 测试入口（不需要改）
├── reports/         ← 报告输出目录
└── logs/            ← 日志输出目录
```

> 💡 **核心操作**：你只需在 `data/` 目录下维护 Excel 用例文件 + 在 `config/` 下维护环境配置，无需写 Python 代码。

## 3. 配置说明

### 3.1 环境配置

编辑 `config/env/test.yaml`（测试环境示例）：

```yaml
base_url: "https://test-api.example.com"    # 被测服务地址

database:
  mysql:
    host: "192.168.1.100"
    port: 3306
    user: "root"
    password: "${DB_PASSWORD}"               # 从环境变量注入
    database: "test_db"
```

**多环境切换**：框架自带 `dev.yaml` / `test.yaml` / `prod.yaml` 三个环境模板，按需配置各环境的 `base_url` 和数据库地址即可。

### 3.2 全局配置

编辑 `config/settings.yaml` 调整框架行为：

```yaml
timeout:
  http_request: 30        # HTTP 请求超时秒数，可按接口响应速度调整
  ui_action: 10000        # UI 操作超时毫秒，慢页面可调大
  ui_navigation: 30000    # 页面加载超时毫秒

retry:
  max_attempts: 2         # 失败自动重试次数
  delay: 1000             # 重试间隔毫秒

logging:
  level: "INFO"           # DEBUG 可输出更多细节
```

### 3.3 浏览器配置

编辑 `config/browser.yaml`：

```yaml
browser:
  type: "chromium"        # chromium / firefox / webkit
  headless: false         # false=可见浏览器; true=无头模式(CI用)
  slow_mo: 100            # 操作减速毫秒，调试时有用
  viewport:
    width: 1920
    height: 1080
```

### 3.4 敏感信息处理

**绝对不要在 YAML 中写明文密码**，使用环境变量注入：

```bash
# Linux/macOS
export DB_HOST=192.168.1.100
export DB_PASSWORD=MySecret123

# Windows PowerShell
$env:DB_HOST="192.168.1.100"
$env:DB_PASSWORD="MySecret123"

# Windows CMD
set DB_HOST=192.168.1.100
set DB_PASSWORD=MySecret123
```

## 4. 编写测试用例

> 详细模板参考 [测试用例模板.md](./测试用例模板.md)

### 4.1 API 用例快速上手

在 `data/api/` 下新建 Excel 文件，比如 `01_用户模块.xlsx`：

| case_id | case_name | method | url | body | assertions | tags |
|---|---|---|---|---|---|---|
| API_USER_001 | 正常登录 | POST | /api/v1/login | {"username":"admin","password":"123456"} | [{"type":"status_code","value":200},{"type":"jsonpath","key":"$.code","value":"0"}] | smoke,p0 |

**最少必填字段**：`case_id`、`case_name`、`method`、`url`、`assertions`

### 4.2 UI 用例快速上手

在 `data/ui/` 下新建 Excel 文件，比如 `01_登录页.xlsx`：

| case_id | case_name | page_url | steps | assertions | tags |
|---|---|---|---|---|---|
| UI_LOGIN_001 | 正常登录 | {{base_url}}/login | [{"action":"fill","selector":"#username","value":"admin"},{"action":"fill","selector":"#password","value":"123456"},{"action":"click","selector":"button[type='submit']"}] | [{"type":"url_contains","value":"/dashboard"}] | smoke,p0 |

**最少必填字段**：`case_id`、`case_name`、`page_url`、`steps`、`assertions`

### 4.3 变量引用

| 写法 | 来源 | 作用域 |
|---|---|---|
| `{{token}}` | 前置用例 `extract` 提取的变量 | SESSION |
| `{{admin_username}}` | YAML 配置 `custom.admin_username` | GLOBAL |
| `${timestamp()}` | 内置函数，实时生成 | — |
| `${uuid()}` | 内置函数，实时生成 | — |

示例：在 `body` 中使用变量生成唯一测试数据：

```json
{"name": "test_${timestamp()}", "email": "${uuid()}@test.com"}
```

### 4.4 用例依赖

下游用例可通过 `depends_on` 引用上游的 `case_id`，自动获取上游提取的变量：

```
API_USER_002 (depends_on=API_USER_001)
  → 自动继承 API_USER_001 提取的 {{token}}
```

### 4.5 断言配置

断言是多条组合的 JSON 数组，常用断言类型：

```json
[
  {"type": "status_code", "value": 200},                     ← HTTP 状态码
  {"type": "jsonpath", "key": "$.code", "value": "0"},       ← JSONPath 比对
  {"type": "jsonpath", "key": "$.data.token", "not_null": true}, ← 非空校验
  {"type": "contains", "value": "success"},                   ← 响应包含文本
  {"type": "response_time", "max_ms": 3000}                   ← 响应时间
]
```

## 5. 执行测试

### 5.1 命令行速查

```bash
# ═══════════════════════════════════════════
# API 测试
# ═══════════════════════════════════════════

# 执行全部 API 用例
pytest test_cases/test_api_executor.py --env=test --alluredir=reports/allure-results -v

# 执行指定 Excel 文件
pytest test_cases/test_api_executor.py --case-file=data/api/01_用户模块.xlsx --env=test -v

# 按标签过滤（只跑冒烟测试）
pytest test_cases/test_api_executor.py --env=test -m smoke -v

# 按多个标签过滤
pytest test_cases/test_api_executor.py --env=test -m "smoke and p0" -v

# ═══════════════════════════════════════════
# UI 测试
# ═══════════════════════════════════════════

# 执行全部 UI 用例（可见浏览器）
pytest test_cases/test_ui_executor.py --env=test --browser=chromium --alluredir=reports/allure-results -v

# 无头模式执行（CI/CD 环境）
pytest test_cases/test_ui_executor.py --env=test --browser=chromium --headless -v

# 按标签过滤
pytest test_cases/test_ui_executor.py --env=test -m smoke --headless -v

# ═══════════════════════════════════════════
# 全部测试
# ═══════════════════════════════════════════

# 执行所有 API + UI
pytest test_cases/ --env=test --alluredir=reports/allure-results -v

# 只跑 p0 优先级的全部用例
pytest test_cases/ --env=test -m p0 -v

# 排除慢速用例
pytest test_cases/ --env=test -m "not slow" -v
```

### 5.2 并行执行

```bash
# 4 个 worker 并行（适合用例数多时加速）
pytest test_cases/ -n 4 --dist=loadscope --env=test --alluredir=reports/allure-results -v

# 自动检测 CPU 核心数
pytest test_cases/ -n auto --env=test --alluredir=reports/allure-results -v
```

### 5.3 失败重跑

```bash
# 失败用例自动重跑 2 次
pytest test_cases/ --reruns 2 --env=test --alluredir=reports/allure-results -v
```

### 5.4 调试模式

```bash
# 只执行第一个失败的用例就停止
pytest test_cases/ --env=test -x

# 显示最详细的输出
pytest test_cases/ --env=test -vv --tb=long

# 进入 pdb 调试（失败后进入交互式调试）
pytest test_cases/ --env=test --pdb
```

## 6. 查看报告

### 6.1 生成 Allure 报告

```bash
# 执行后生成 HTML 报告
allure generate reports/allure-results -o reports/allure-reports --clean

# 打开报告（自动启动本地 Web 服务）
allure open reports/allure-reports
```

### 6.2 报告内容

Allure 报告包含以下信息：

| 面板 | 内容 |
|---|---|
| **Overview** | 通过率、持续时间、严重程度分布 |
| **Suites** | 按模块/Sheet 分组展示用例 |
| **Behaviors** | 按功能标签（smoke/regression）分组 |
| **Graphs** | 通过率趋势图 |
| **Timeline** | 执行时间线（并行执行时有用） |
| **Environment** | 环境、分支、提交哈希等信息 |

### 6.3 单用例报告详情

点击单个用例可查看：

- ✅ 每个步骤的通过/失败状态
- ✅ 请求详情（Method / URL / Headers / Body）
- ✅ 响应详情（Status Code / Body / Response Time）
- ✅ 断言详情（期望值 vs 实际值）
- ✅ 失败截图（UI 用例）
- ✅ 执行日志

## 7. 高级功能

### 7.1 自定义钩子

#### 前置钩子示例：自动登录

在 `hooks/pre_hooks.py` 中定义：

```python
def login_as_admin(client, variable_manager, config):
    """管理员登录，获取 token 并注入到后续请求"""
    resp = client.post("/api/v1/login", json={
        "username": config["custom"]["admin_username"],
        "password": config["custom"]["admin_password"]
    })
    token = resp.json()["data"]["token"]
    variable_manager.set("token", token)
    client.headers["Authorization"] = f"Bearer {token}"
```

在 Excel 用例的 `pre_hook` 列填写：

```
login_as_admin
```

#### 后置钩子示例：数据清理

在 `hooks/post_hooks.py` 中定义：

```python
def cleanup_user(client, variable_manager, db):
    """清理测试创建的用户"""
    user_id = variable_manager.get("user_id")
    if user_id:
        db.execute(f"DELETE FROM users WHERE id = {user_id}")
```

在 Excel 用例的 `post_hook` 列填写：

```
cleanup_user
```

### 7.2 数据库断言

Excel 断言列中直接写 SQL：

```json
[
  {"type": "status_code", "value": 200},
  {"type": "db", "query": "SELECT status FROM users WHERE id='{{user_id}}'", "value": "active"}
]
```

### 7.3 自定义函数注册

`core/variable_manager.py` 中注册：

```python
VariableManager.register_function("today", lambda: datetime.now().strftime("%Y-%m-%d"))
```

Excel 中使用：

```
{{body}} → {"date": "${today()}"}
```

### 7.4 环境变量注入

在 YAML 中使用 `${VAR:default}` 语法：

```yaml
database:
  password: "${DB_PASSWORD}"              # 必填，无默认值时报错
  host: "${DB_HOST:localhost}"            # 优先取环境变量，否则用 localhost
```

### 7.5 CI/CD 接入

#### Jenkins Pipeline

```groovy
stage('自动化测试') {
    steps {
        sh '''
            pip install -r requirements.txt
            playwright install chromium --with-deps
            pytest test_cases/ --env=test --headless --alluredir=reports/allure-results -v
        '''
    }
    post {
        always {
            allure includeProperties: false, results: [[path: 'reports/allure-results']]
        }
    }
}
```

#### GitHub Actions

```yaml
- name: Run Tests
  env:
    DB_PASSWORD: ${{ secrets.DB_PASSWORD }}
  run: |
    pip install -r requirements.txt
    playwright install chromium --with-deps
    pytest test_cases/ --env=test --headless --alluredir=reports/allure-results -v
```

## 8. 常见问题

### Q1: 执行报错 `unknown marker: smoke`

**原因**：pytest.ini 中未注册该 marker。

**解决**：确认 `pytest.ini` 中存在对应 marker：

```ini
markers =
    smoke: 冒烟测试
```

### Q2: 变量 `{{token}}` 未解析

**原因**：变量未被前置用例提取到变量池。

**排查**：
1. 确认上游用例的 `extract` 列正确配置（如 `token=$.data.token`）
2. 确认下游用例的 `depends_on` 指向了上游 `case_id`
3. 确认上游用例确实执行成功（提取发生在成功响应后）

### Q3: UI 用例超时找不到元素

**原因**：页面加载慢，`wait_for_selector` 默认超时不够。

**解决**：
- 在步骤中显式加 `{"action":"wait","value":"3000"}`
- 调大 `config/settings.yaml` 中的 `timeout.ui_action`
- 检查 selector 是否正确（打开浏览器 F12 确认）

### Q4: Playwright 报错 `Browser closed unexpectedly`

**原因**：Chromium 缺少系统依赖。

**解决**：

```bash
playwright install-deps chromium
```

### Q5: Excel 中的 JSON 解析失败

**常见错误**：
- 使用了单引号 `{'key':'value'}` → 必须改为双引号 `{"key":"value"}`
- JSON 中有换行 → 必须写成紧凑一行
- 中文引号混入 → 检查是否粘贴了 `"中文引号"`

### Q6: Allure 报告为空

**原因**：执行时未指定 `--alluredir` 参数，或目标目录权限不足。

**解决**：

```bash
# 先清理旧结果
rm -rf reports/allure-results/*

# 执行时必须加 --alluredir
pytest test_cases/ --env=test --alluredir=reports/allure-results -v

# 确认结果文件存在
ls reports/allure-results/
# 应看到 *.json 文件
```

### Q7: 并行执行时变量串了

**原因**：使用了 GLOBAL 作用域的变量，被多个线程同时读写。

**解决**：
- 需要跨用例共享的变量用 `SESSION` 作用域
- 用例内部用 `CASE` 作用域
- 使用 `${uuid()}` / `${timestamp()}` 确保数据唯一性

## 9. 附录

### 9.1 CLI 参数完整列表

| 参数 | 说明 | 默认值 |
|---|---|---|
| `--env` | 环境名称 | `test` |
| `--browser` | 浏览器类型 | `chromium` |
| `--headless` | 无头模式 | `false` |
| `--alluredir` | Allure 结果目录 | — |
| `--case-file` | 指定用例文件 | 全部 |
| `--case-dir` | 指定用例目录 | `data/api/` 或 `data/ui/` |
| `-m` | marker 过滤 | — |
| `-n` | 并行 worker 数 | 1（串行） |
| `--reruns` | 失败重试次数 | 0 |
| `-x` | 首个失败即停止 | — |
| `-k` | 用例名称关键字过滤 | — |
| `-v` / `-vv` | 输出详细程度 | — |

### 9.2 断言类型完整参考

| type | 适用 | 参数 | 示例 |
|---|---|---|---|
| `status_code` | API | `value` | `{"type":"status_code","value":200}` |
| `jsonpath` | API | `key`, `value` 或 `not_null` | `{"type":"jsonpath","key":"$.code","value":"0"}` |
| `contains` | API | `value` | `{"type":"contains","value":"success"}` |
| `response_time` | API | `max_ms` | `{"type":"response_time","max_ms":3000}` |
| `header` | API | `key`, `value` | `{"type":"header","key":"Content-Type","value":"application/json"}` |
| `schema` | API | `schema` (JSON Schema) | `{"type":"schema","schema":{"type":"object"}}` |
| `db` | API | `query`, `value` | `{"type":"db","query":"SELECT count(*) FROM users","value":1}` |
| `soft` | API | 同 jsonpath + `comment` | `{"type":"soft","key":"$.optional","value":"x","comment":"非关键"}` |
| `element_visible` | UI | `target` (选择器) | `{"type":"element_visible","target":".success-msg"}` |
| `text_equals` | UI | `target`, `value` | `{"type":"text_equals","target":"h1","value":"欢迎"}` |
| `url_contains` | UI | `value` | `{"type":"url_contains","value":"/dashboard"}` |
| `page_title` | UI | `value` | `{"type":"page_title","value":"控制台"}` |
